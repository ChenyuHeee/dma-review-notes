## 2. Generating Permutations (生成排列)

### 2.1 Problem

> List (generate) all permutations of any set with $n$ elements.

**Approach:**
1. 将任意 $n$ 元素集合与 $\{1, 2, \ldots, n\}$ 建立一一对应
2. 生成 $\{1, 2, \ldots, n\}$ 的所有排列
3. 将整数映射回原集合的元素

### 2.2 Lexicographic Order (字典序)

> **Lexicographic ordering** (字典序): The permutation $a_1 a_2 \ldots a_n$ **precedes** $b_1 b_2 \ldots b_n$ if for some $k$ ($1 \le k \le n$):
> - $a_1 = b_1, a_2 = b_2, \ldots, a_{k-1} = b_{k-1}$
> - $a_k < b_k$

**通俗理解:** 和字典排序一样 — 从左到右逐位比较，遇到第一位不同的数字，较小的那个排列在前。

**Example:**
```
123465  precedes  124635
```
比较过程: 前两位 12 相同，第三位 3 < 4，所以 123465 排在 124635 之前。

### 2.3 Algorithm: Finding the Next Permutation

**Goal:** Given a permutation $a_1 a_2 \ldots a_n$, find the **next larger permutation** in lexicographic order.

**Algorithm:**

1. **Find the largest $j$** such that $a_j < a_{j+1}$  (从右向左找第一个递增相邻对)
   - 此时 $a_{j+1} > a_{j+2} > \cdots > a_n$ (后缀是递减的)

2. **Find the smallest integer** among $a_{j+1}, a_{j+2}, \ldots, a_n$ that is **greater than** $a_j$
   - 即后缀中比 $a_j$ 大的最小元素

3. **Swap** $a_j$ with that element

4. **Reverse** (or sort in increasing order) the suffix $a_{j+1}, a_{j+2}, \ldots, a_n$

**直观理解:** 找到最后一个可以"增大的位置"，将该位置的数换成后缀中稍大的数，然后将后缀反转为最小顺序。

### 2.4 Example

> What is the next permutation after **124653** in lexicographic order?

**Step-by-step:**

1. 从右向左找 $a_j < a_{j+1}$:
   - 65: 6 > 5 ✗
   - 46: 4 < 6 ✓ → $j = 3$, $a_j = 4$
   - 后缀 = 653 (递减)

2. 在后缀 653 中找比 4 大的最小数: **5**

3. Swap 4 和 5 → **125643**

4. 反置后缀 643 → **346**
   - (实际是排序后缀为递增: 346)

**Result:** 125346

**Verification:** 124653 → 125346 → 125364 → 125436 → ...

## 3. Generating Combinations (生成组合)

### 3.1 Problem 1: Generate All Subsets

> Generate all combinations (subsets) of a finite set.

**Approach:**
- 用长度为 $n$ 的 **bit string** 表示子集: 第 i 位 = 1 表示包含第 i 个元素
- 所有 $2^n$ 个 bit strings 按 **二进制递增** 顺序列出

**Algorithm:**
1. Start with $000\ldots00$ (n zeros)
2. Repeatedly find the next binary expansion until $111\ldots11$

**How to find the next bit string:**
> Locate the **first position from the right** that is **not a 1**, change it to **1**, and change **all 1s to its right** to **0**.

**Example:**
```
1000110011  →  1000110100
```
- 从右向左找第一个 0: 位置 (从右数第 3 位)
- 该位 0→1
- 右侧所有 1→0

### 3.2 Problem 2: Generate All r-Combinations

> Generate all r-combinations of the set $\{1, 2, \ldots, n\}$.

**Key Idea:** 将每个 r-combination 表示为递增序列 $a_1 < a_2 < \cdots < a_r$。

**Algorithm:**
1. Start with the smallest r-combination: $\{1, 2, \ldots, r\}$
2. Given current combination $\{a_1, a_2, \ldots, a_r\}$, find the next:

   - Find the **last** element $a_i$ such that $a_i \neq n - r + i$
     - (即可以递增而不超出范围的最右元素)
   - Replace $a_i$ with $a_i + 1$
   - For $j = i+1, \ldots, r$: set $a_j = a_{i} + (j - i)$
     - (将后面的元素设为最小的连续递增序列)

#### Example

> Given $S_i = \{2, 3, 5, 6, 9, 10\}$ (a 6-combination from {1..10}), find $S_{i+1}$.

**Solution:**
- 检查每个位置的最大可能值 (n - r + i):
  - $a_1=2$, max $=10-6+1=5$ → OK
  - $a_2=3$, max $=10-6+2=6$ → OK
  - $a_3=5$, max $=10-6+3=7$ → OK
  - $a_4=6$, max $=10-6+4=8$ → OK
  - $a_5=9$, max $=10-6+5=9$ → $a_5 = 9 = 9$ (已达上限)
  - $a_6=10$, max $=10-6+6=10$ → $a_6 = 10 = 10$ (已达上限)

- 从右向左找最后一个 $a_i \neq n-r+i$: $a_4 = 6 \neq 8$ → $i=4$

- $a_4 \leftarrow 6+1 = 7$
- 后面接最小递增: $a_5 = 8, a_6 = 9$

**Result:** $S_{i+1} = \{2, 3, 5, 7, 8, 9\}$

## 4. 考点提示 (Exam Tips)

1. **Lexicographic Order 定义:** 按字典序比较排列 — 从左到右逐位比较
2. **Next Permutation 算法:**
   - 从右向左找 $a_j < a_{j+1}$
   - 在后缀中找比 $a_j$ 大的最小数
   - 交换 + 反置后缀
3. **Next Bit String 算法:** 从右找第一个 0 → 改 1，右侧清 0
4. **Next r-Combination 算法:**
   - 从右向左找最后一个可以增大的位置 ($a_i \neq n-r+i$)
   - 该位 +1，后面置为最小递增序列
5. 这些算法在考试中可能以 **手工演算** 的形式出现，理解步骤比背代码更重要
