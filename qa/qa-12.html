Huffman Coding（哈夫曼编码）

2026-06-27

问题背景

有 6 个符号 A, B, C, D, E, F，各自出现频率不同。用二进制给每个符号编码，怎样让总比特数最少？

ABCDEF
15%25%5%20%10%25%

定长码：6 个符号需要 $\lceil \log_2 6 \rceil = 3$ 位。Huffman 能做到更少（约 2.45 位/符号）。

Huffman 算法：贪心合并最小的两个

核心思想：出现次数多的给短编码，出现次数少的给长编码。每次把频率最小的两个合并成一个新节点。

初始（6 个叶子节点，各带频率）：

A:15, B:25, C:5, D:20, E:10, F:25

Step 1：最小的两个是 C(5) 和 E(10)，合并 → CE = 15

Step 2：现在有 A(15), CE(15), B(25), D(20), F(25)。最小的两个是 A(15) 和 CE(15)？不，D(20) 和 A(15) 比。实际上最小是 A(15) 和 CE(15)，合并 → A-CE = 30

Step 3：现在有 A-CE(30), B(25), D(20), F(25)。最小的两个是 D(20) 和 F(25)？B(25)=F(25)，D(20) 最小。D(20) 和 F(25) 合并 → DF = 45

Step 4：现在有 A-CE(30), B(25), DF(45)。最小的两个是 B(25) 和 A-CE(30)？实际是 B(25) 和 A-CE(30)，合并 → B-A-CE = 55

Step 5：最后 B-A-CE(55) 和 DF(45) 合并 → 根 = 100

注意：同频率时选哪个都可以，合并结果可能不同，但平均位数相同。

从合并树生成编码

合并过程形成了一棵二叉树。从根出发，左走 = 0，右走 = 1：

ROOT(1.00) / \ 0 1 / \ B-A-CE(0.55) DF(0.45) / \ / \ 0 1 0 1 / \ / \ B(0.25) A-CE(0.30) D(0.20) F(0.25) / \ 0 1 / \ A(0.15) CE(0.15) / \ 0 1 / \ C(0.05) E(0.10)

从根到每个叶子的路径就是编码：

符号路径编码长度频率
B0000225%
A010010315%
C0110011045%
E01110111410%
D1010220%
F1111225%

频率高的（B, D, F）用 2 位短码，频率低的（C, E）用 4 位长码。

计算平均比特数

$$2 \times (0.25 + 0.20 + 0.25) + 3 \times 0.15 + 4 \times (0.05 + 0.10) = 1.40 + 0.45 + 0.60 = 2.45$$

定长码需要 3 位，Huffman 只需要 2.45 位，节省约 18%。

算法步骤（背这个）

1 所有符号各自作为一个节点，频率为权重
2 反复：选频率最小的两个节点，合并成一个新节点，频率 = 两者之和
3 只剩一个节点时停止（即树的根）
4 从根到每个叶子的路径：左分支 = 0，右分支 = 1，路径上的 01 串即为编码
考试常见考法

给一张频率表 → 画出合并树 → 写出编码 → 算平均位数 → 和定长码比较。全程手算。
